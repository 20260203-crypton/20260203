## AI와의 대화
Q!. 상대경로와 절대경로에 대해서 자세하게 설명.

AI 답변: 절대 경로는 최초의 시작점부터 해당 파일까지 이르는 전체 경로를 다 적는 방식.  
         어디서든 그 경로를 누르면 항상 똑같은 파일을 가리킴.  
         하지만 내 컴퓨터에서는 잘 작동하더라도 다른사람에게 보내거나 서버에 올리면 경로가 바뀌기에 작동하지 않을 수 있음.  
         예시) C:\Users\Documents\photos\cat.jpg  

           상대 경로는 현재 내가 있는 위치를 기준으로 파일의 위치를 나타내는 방식.  
           현재 작업 중인 위치가 바뀌면 상대 경로도 수정해야함.  
           하지만 폴더의 내부 구조만 동일 하다면 다른곳에 옮기더라도 잘 작동한다.  
           주요 기호 - . = 현재 위치 의미   .. = 한 단계 위 부모 폴더로 나가는 것 의미   / = 폴더 내부로 들어갈 때 사용  
           예시) ../images/dog.png : 한 칸 밖으로 나가서 images 폴더 안에 있는 dog.png를 찾아라.     ./main.css : 현재 폴더에 있는 main.css 파일을 찾아라.  
           
           팀 프로젝트 시에는 상대경로를 주로 이용  

Q2. 스프라이트 파일에서 캐릭터마다 sprite sheet.png 가 있는데 이 파일들이 어떤 캐릭터의 스프라이트 시트인지 컴퓨터가 어떻게 파악하는거야?  

AI 답변: 파일에 이름을 추가해서 구분시켜서 그 이름을 쓰거나  
         캐릭터별로 파일을 만들어 그 안에 넣어놨다면 상대 경로를 이용해 파악할 수 있음.  

Q3. base64 임베딩이 뭐야?  

AI 답변: 이미지나 파일같은 데이터를 글자 형태로 바꿔서 텍스트 파일 안에 직접 박아 넣는 것.  
         이 base64 임베딩은 데이터가 전송 중에 깨질 수 있는 환경에서 텍스트 형태로 안전하게 전달 할 수 있고 소스코드 자체가 데이터를 품고있기에 파일 경로 문제를 신경 쓸 필요가 없다.   
         하지만 용량이 증가하고 가독성이 저하된다는 단점도 존재한다.
