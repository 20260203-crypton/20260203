## 관찰 내용



### 원형 vs AABB
Pygame으로 800x600 창을 만들어줘.
회색 사각형 오브젝트 2개를 그려줘.
하나는 방향키로 이동 가능, 하나는 화면 중앙 고정.
두 오브젝트의 AABB (빨간 사각형)를 함께 표시해줘1. 현재 코드에 원형 Bounding Box를 추가해줘.
각 오브젝트의 중심에서 파란 원을 그려줘.
반지름은 오브젝트 너비의 절반으로 설정해줘.
원형 충돌 감지도 구현하고, 충돌 시 화면 배경을 노란색으로 바꿔줘.
를 AI에게 요구하여 코드를 받음.

이를 이용해 관찰한 결과
-AABB가 먼저 충돌하는 경우 - 원형은 사각형에 내접하도록 그렸기에 사각형의 모서리 부분에서 AABB만 충돌하는 경우가 일어난다.
-AABB보다 원형이 먼저 충돌하는경우 - 없었다 원이 사각형에 완전히 내접해 있기 때문이다.
-두 방식이 동시에 충돌하는 경우 - 원이 사각형에 내접하기에 그 접점에서 동시에 충돌하게된다.

관찰질문
오브젝트가 정삭각형일 때 원형 Bounding Box의 모서리 부분에서 어떤 일이 일어나는가? - 충돌이 감지되지 않는다 
왜 그런 현상이 일어나는가? - 원의 반지름을 사각형의 너비의 절반으로 했다. 따라서 원은 사각형에 내접하게 된다. 
                          그러므로 사각형에는 원에 포함되지 않는 남는 부분이 생기게 된다. 이곳은 원과 겹치지 않기에 충돌이 일어나지 않게된다.
### AABB vs OBB
-정사각형 오브젝트 회전없이 접근 - 회전이 없기에 OBB = AABB 이다. 모서리로 접근하지 않는다면 원형도 같을것이다.
-모서리 접근 - AABB = OBB는 같았다. 허나 원형은 오차가 발생한다.
-회전 중 접근 - OBB는 회전을 반영하기에 오차가 발생하지 않는다. 허나 AABB는 회전을 반영하지 않기 때문에 오차가 발생하게된다.

## AI와의 대화에서 배운것
AABB는 처리가 빠르지만 부정확할때가 있고 OBB는 정확하지만 계산량이 많아 처리가 느리다는것을 알게 되었다.
원형 충돌 감지에서 math.sqrt를 쓰는 이유: 원형 충돌은 기본적으로 두 점 사이의 거리를 구하는 과정이다. 그래서 코드에서 distance = math.sqrt(dx*dx + dy*dy)로 거리를 구하고
                                        distance < (r1 + r2)로 반지름 합보다 가까우면 충돌한것으로 판정한다.
그런데 sqrt없이도 가능하다고 한다 비교만 하면 되기 때문에 √(dx² + dy²) < R 의 양변에 제곱을 하여 dx² + dy² < R²로 바꾸고 비교하는것이다.
sqrt를 사용하면 진짜 거리를 재는것이기에 사람이 이해하기 쉽지만 계산이 느리고 sqrt를 사용하지 않으면 덜 직관적이지만 수학적으로는 같고 계산이 빠르다고 한다.
따라서 게임에서는 속도 때문에 sqrt 없이 제곱하여 비교한다고 한다.
