## resource_path() 를 써야 하는 이유

resource_path()는 PyInstaller로 파이썬 프로그램을 .exe 파일(단일 파일 빌드)로 만들 때 발생하는 경로 인식 문제를 해결하기 위해 사용

## 빌드 명령어
pyinstaller --onefile --noconsol --add-data "sound;sound" immunoid_final.py

스프라이트는 문자열로 바꿔서 넣었기에 사운드만 해줬음.

## AI 활용 내역

Q1. PyInstaller의 작동 원리가 뭐야

AI답변 :
PyInstaller는 파이썬 스크립트가 파이썬이 설치되지 않은 환경에서도 실행될 수 있도록 모든 의존성(인터프리터, 라이브러리, 소스 코드)을 하나의 패키지로 묶어주는 번들링(Bundling) 도구입니다.
작동 원리는 크게 분석(Analysis), 병합(Aggregation), 그리고 실행(Execution) 세 단계로 나뉩니다.

1. 분석 및 수집 (Analysis & Collection)
가장 먼저 PyInstaller는 작성한 .py 파일을 훑어서 어떤 라이브러리가 필요한지 찾아냅니다.
2. 패키징 방식 (One-Dir vs One-File)
이 수집된 파일들을 어떻게 구성하느냐에 따라 두 가지 모드로 나뉩니다.
① One-Directory (-D)
모든 파일이 하나의 폴더 안에 들어있는 형태입니다. 실행 파일(.exe) 외에 수많은 .dll 파일들이 노출됩니다.

장점: 프로그램 실행 속도가 빠릅니다. (압축을 풀 필요가 없기 때문)

② One-File (-F)
모든 라이브러리와 인터프리터를 하나의 .exe 파일 안에 압축해서 넣습니다.
3. 실행 시점의 동작 (Runtime Process)
PyInstaller로 만든 .exe 파일은 실제 "컴파일된 기계어 코드"가 아니라, "압축된 파이썬 환경의 압축 해제기"에 가깝습니다. 특히 One-File 모드일 때의 동작이 중요합니다.

Q2. sound나 sprite같은 에셋이 없으면 pyinstaller 파일.py만으로도 .exe파일이 만들어지고 실행도 가능해?

AI답변 :
네, 가능합니다!

결론부터 말씀드리면, 별도의 에셋(Sound, Sprite 등)이 없는 순수 파이썬 코드(file.py)라면 추가 설정 없이 pyinstaller file.py 명령어만으로 완벽하게 작동하는 .exe 파일을 만들 수 있습니다.

구체적인 이유는 다음과 같습니다.

1. 코드 자체는 '바이트코드'로 포함됨
파이썬 스크립트 내에 작성한 모든 로직, 함수, 클래스는 PyInstaller가 분석하여 컴파일된 형태(.pyc)로 실행 파일 안에 집어넣습니다. 즉, 소스 코드만 있는 프로그램은 실행 파일 하나가 곧 프로그램 전체가 됩니다.

2. 표준 라이브러리와 외부 모듈의 포함
코드 상단에 import math, import os 혹은 pip로 설치한 import requests 같은 모듈이 있다면, PyInstaller는 이 모듈들을 찾아내어 실행 파일 안에 함께 패키징합니다. 사용자의 컴퓨터에 파이썬이 깔려 있지 않아도 실행되는 이유가 바로 이것입니다.

Q3. .spec이랑 .dist은 뭐야

AI답변 : 
PyInstaller로 빌드를 시작하면 평소 보지 못했던 폴더와 파일들이 생기는데, 각각 '설계도'와 '완성품'이라고 생각하면 이해가 쉽습니다.

1. .spec 파일 (Specification: 설계도)
pyinstaller main.py를 실행하면 가장 먼저 생성되는 텍스트 파일입니다.

역할: PyInstaller가 프로그램을 어떻게 빌드해야 할지 적혀 있는 명세서입니다.

주요 내용: * 어떤 스크립트를 실행 파일로 만들 것인가?

함께 묶어야 할 데이터 파일(이미지, 사운드 등)은 무엇인가?

아이콘 파일은 어디에 있는가?

활용법: 처음에는 자동으로 만들어지지만, 나중에 설정이 복잡해지면(파일 수백 개를 추가해야 하거나 특수한 라이브러리를 쓸 때) 이 파일을 직접 메모장으로 열어 수정하기도 합니다. 수정 후에는 pyinstaller main.spec이라고 입력해 빌드할 수 있습니다.

2. dist 폴더 (Distribution: 배포 폴더)
빌드가 성공적으로 끝나면 생성되는 폴더로, 우리가 최종적으로 얻으려는 결과물이 들어있습니다.

역할: 사용자에게 전달할 최종 배포용 파일이 저장되는 곳입니다.

특징:

--onefile 옵션을 썼다면 이 안에 달랑 .exe 파일 하나만 들어있을 겁니다.

이 폴더 안에 있는 파일만 챙겨서 친구에게 주면 됩니다. (다른 build, __pycache__ 같은 폴더는 줄 필요 없습니다.)
